VSCode 函数列表插件

1. 核心目标

我想把 Keil IDE 的“函数标签页”体验搬到 VSCode 中。最终效果是：在侧边栏看到一个当前工程内所有函数的列表，双击任一函数就能直接跳转到源码定义处，方便快速查看参数和实现。

2. 功能需求（按优先级排列）

P0 - 必须实现

· 全局函数扫描：能遍历工作区所有源码文件，提取出所有函数/方法定义。
· 列表展示：以列表形式展示在 VSCode 侧边栏的独立视图中。
· 双击跳转：点击列表中任意一个函数，编辑器自动打开对应文件，并定位到该函数的定义行。
· 实时搜索过滤：列表顶部有搜索框，输入关键字可以即时过滤函数名。

P1 - 重要优化

· 多语言支持（通过配置文件切换）：
  我希望插件的行为可以通过 settings.json 配置，为不同语言设置不同的展示模式和解析规则。一个典型的配置示例如下：
  ```json
  "functionList.profiles": {
    "c": {
      "viewMode": "flat",
      "functionPattern": "..."
    },
    "java": {
      "viewMode": "structured",
      "classPattern": "...",
      "methodPattern": "..."
    }
  }
  ```
  · viewMode 为 flat 时，列表直接平铺所有函数（适合 C 语言）。
  · viewMode 为 structured 时，列表按“文件 -> 类 -> 方法”的树形结构展开（适合 Java/Kotlin）。
  · 不同 profile 可以自定义用于提取函数/类的正则表达式。
· 列表项格式：在结构化模式下，方法显示为 ClassName.methodName；平铺模式下直接显示 functionName。鼠标悬停时最好能显示文件路径和简易签名。

P2 - 锦上添花

· 手动刷新按钮：列表标题栏有一个刷新图标，点击后重新扫描整个工程。
· 自动监听文件保存：文件保存时，自动更新列表中该文件对应的函数列表。
· 排除目录：默认忽略 node_modules、.git 等目录，最好也能配置。

3. 技术建议（不强制，供参考）

· 推荐使用 vscode.workspace.findFiles 来获取工作区文件列表。
· 推荐使用 TreeDataProvider 来构建视图，配合 enableFindWidget 属性实现原生搜索框。
· 函数解析可以先用正则匹配实现，不必上完整的 AST 解析器，优先保证轻量和响应速度。
